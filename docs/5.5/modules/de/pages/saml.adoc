== SAML-IDP-Konfiguration

Um {product-name} für die Verwendung des Okta SAML IDP-Servers zu konfigurieren, konfigurieren Sie zunächst den SAML IDP-Server auf der Okta-Website.

image:okta.png[Okta]

. Kopieren Sie "SAML Redirect URI" von der Seite {product-name} SAML Setting und fügen Sie sie in die Felder Single Sign On Url, Receiver Url und Destination Url von Okta ein.
. Verschlüsselung der Behauptung: Dieses Feld muss *unverschlüsselt* sein *.*
. Attribut-Aussagen: Geben Sie die Attribute E-Mail und Benutzername ein.
. Gruppen-Attribut-Anweisungen: Aktivieren Sie diese Option, wenn eine gruppenbasierte Rollenzuordnung erforderlich ist. Der Standardattributname, nach dem {product-name} sucht, ist NVRoleGroup. Wenn ein anderer Attributname für die Gruppenzugehörigkeit des Benutzers verwendet wird, kann dieser auf der SAML-Einstellungsseite von {product-name} angepasst werden.

Konfigurieren Sie die SAML-Einstellungen in der {product-name} UI-Konsole.

image:saml1.png[SAML]

Verwenden Sie die Schaltfläche "View Setup Instructions" (Setup-Anweisungen anzeigen), wie im folgenden Screenshot gezeigt, um die folgenden Informationen zu finden, und kopieren Sie sie in die SAML-Seite von {product-name}.

. Identity Provider Single Sign-On URL
. Identitätsanbieter Emittent
. X.509-Zertifikat
. Geben Sie den Namen des Gruppenattributs an, wenn ein anderer Wert als der Standardwert verwendet wird.

image:setting.png[Einstellung]

Nachdem der Benutzer authentifiziert ist, kann die richtige Rolle mit der gruppenbasierten Rollenzuordnungskonfiguration abgeleitet werden. So richten Sie eine gruppenbasierte Rollenzuordnung ein,

. Wenn die gruppenbasierte Rollenzuordnung nicht konfiguriert ist oder die entsprechenden Gruppen nicht gefunden werden können, wird dem authentifizierten Benutzer die Standardrolle zugewiesen. Wenn die Standardrolle auf Keine gesetzt ist, kann sich der Benutzer nicht anmelden, wenn die gruppenbasierte Rollenzuordnung fehlschlägt.
. Geben Sie eine Liste von Gruppen in den Rollen Admin und Reader an. Das Gruppenattribut des Benutzers wird in der Antwort huckepack genommen, nachdem der Benutzer authentifiziert wurde. Wenn die passende Gruppe gefunden wird, wird dem Benutzer die entsprechende Rolle zugewiesen.
