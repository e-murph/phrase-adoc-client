== Importation d'un fichier au format CRD à partir de la console ou de l'API

{product-name} permet d'importer un fichier formaté CRD à partir de la console. Cependant, ce n'est pas la même chose que de l'appliquer dans Kubernetes en tant que définition de ressource personnalisée (CRD).

Un fichier au format {product-name} CRD peut être importé via la console afin de définir la politique de sécurité (règles) spécifiée dans le fichier. Ces règles ne seront PAS importées en tant que règles désignées "CRD", mais en tant que règles ordinaires "créées par l'utilisateur". Ces règles peuvent donc être modifiées ou supprimées comme les autres règles, à partir de la console ou de l'API. Elles ne sont pas protégées contre les modifications par les règles de la CRD.

Pour importer à partir de la console, allez sur Policy -> Groups et sélectionnez Import Policy Group.

image:4-3_Import_Policy.png[l'importation]

[IMPORTANT]
====
Les règles importées remplacent toutes les règles existantes pour le groupe.
====

Les règles définies à l'aide des fonctions CRD de Kubernetes, par exemple via "kubectl apply my_crd.yaml", créent des règles de type CRD dans {product-name} qui ne peuvent pas être modifiées via la console ou l'API. Ceux-ci ne peuvent être modifiés qu'en mettant à jour le fichier crd et en appliquant le changement via Kubernetes.

Les cas d'utilisation possibles pour l'importation par la console du fichier de règles sont les suivants :

* Configuration initiale (unique) des règles pour un ou plusieurs groupes
* Migration des règles d'un environnement à l'autre
* Création de règles dont la modification doit être autorisée à partir de la console ou de l'API.
