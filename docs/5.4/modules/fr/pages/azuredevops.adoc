== Recherche de vulnérabilités dans le pipeline de construction Azure DevOps

Le scanner {product-name} peut être déclenché à partir du pipeline Azure DevOps en utilisant l'extension {product-name} publiée sur Azure DevOps Marketplace.

image:azure_devops.png[AzureDevOps]

L'extension prend en charge l'analyse à distance et l'analyse locale, le contrôleur {product-name} pouvant analyser à distance une image dans un registre pendant la construction, ou lancer dynamiquement un contrôleur local pour analyser l'image sur la machine virtuelle de l'agent Azure.

En outre, assurez-vous qu'un conteneur de scanner {product-name} est déployé et configuré pour se connecter à l'Allinone ou au contrôleur. Dans les versions 4.0 et ultérieures, le conteneur neuvector/scanner doit être déployé séparément de l'allinone ou du contrôleur.

* Scan image with {product-name} task intègre le scanner de vulnérabilités {product-name} dans un Azure DevOps Pipeline.
* Effectuer des analyses de vulnérabilité d'une image de conteneur après la construction de l'image sur une instance de contrôleur externe {product-name} ou sur une instance de contrôleur locale {product-name} qui s'exécute dans un conteneur de service à l'intérieur d'un pipeline.
* Définir des seuils d'échec de compilation en fonction du nombre de vulnérabilités détectées de différentes sévérités.
* Fournir un rapport détaillé d'un scan d'image pour analyse dans l'onglet résumé de la construction.
* Les instances de contrôleurs externes {product-name} sont définies comme des points d'extrémité de service afin de découpler les définitions du pipeline de construction des paramètres de connexion et des informations d'identification.

Une vue d'ensemble avec des exemples d'écrans est disponible à l'adresse suivante https://marketplace.visualstudio.com/items?itemName=NeuVector.neuvector-vsts.
