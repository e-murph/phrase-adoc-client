== Föderierte Politik

Nachdem ein Master-Cluster erstellt wurde, können im Master-Cluster Regeln für den Verbund erstellt werden, die automatisch an alle Cluster weitergegeben werden. Dies ist nützlich, um globale Regeln zu erstellen, die auf jeden Cluster angewendet werden sollen, wie z. B. globale Netzwerkregeln. Verbundregeln werden in jedem Cluster als schreibgeschützt angezeigt und können vom lokalen Administrator des Clusters NICHT gelöscht oder bearbeitet werden.

Um die Regeln für den Verbund zu konfigurieren, klicken Sie im oberen rechten Dropdown-Menü auf Verbundrichtlinie. Sie sehen Registerkarten für Gruppen, Zugangskontrolle, Netzwerkregeln und andere Regeln, die zusammengeführt werden können. Wählen Sie die Registerkarte und erstellen Sie eine neue Gruppe oder Regel. Im folgenden Beispiel wurden zwei föderierte Gruppen erstellt, die an jeden Cluster weitergegeben werden.

image:fed_group.png[FederatedGroup]

Und die folgende Federated Network Rule wurde erstellt, um den SSL-Zugriff von den Node Demo Pods auf google.com zu ermöglichen.

image:fed_network.png[FederatedNetwork]

Nachdem diese Regeln und Gruppen auf den/die Remote-Cluster übertragen wurden, werden sie als Federated-Regeln und -Gruppen in der Konsole des lokalen Clusters angezeigt.

image:fed_rule_remote.png[FederatedRuleRemote]

Im obigen Beispiel wird die föderierte Regel angezeigt, die sich von den gelernten Regeln und den vom Benutzer erstellten Regeln, die im lokalen Cluster erstellt wurden, unterscheidet. Die vom Benutzer erstellte Regel 1 kann zur Bearbeitung oder Löschung ausgewählt werden, die föderierte Regel hingegen nicht. Darüber hinaus werden die Regeln des föderierten Netzwerks immer an erster Stelle der Liste angezeigt und haben somit Vorrang vor anderen Regeln.

Andere Regeln wie Zulassungskontrolle, Antwort, Prozess und Datei verhalten sich auf die gleiche Weise, mit der Ausnahme, dass die Reihenfolge der Regeln nur für die Netzwerkregeln relevant ist.

Beachten Sie, dass die Konfiguration von Prozess- und Dateiregeln die Auswahl einer Verbundgruppe erfordert, da diese auf eine in der Registerkarte Verbundgruppe definierte Zielgruppe angewendet werden müssen. Nachdem eine neue Gruppe in Federated -> Groups konfiguriert wurde, wird sie als auswählbare Option angezeigt, wenn eine Gruppe in Process oder File rules konfiguriert wird.
