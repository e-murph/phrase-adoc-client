== Konfigurieren von Active Directory

Hier wird erklärt, wie {product-name} sich bei Windows Active Directory authentifiziert.
Die Konfigurationsseite für den Windows Active Directory Server ist unten dargestellt.

image:ad1.png[ad_config]

Name des Benutzers: Dies kann jeder Benutzer sein, der eine Leseberechtigung für das Objekt Base DN hat. Das dn-Attribut sollte wie unten gezeigt verwendet werden, oder der Windows-Anmeldename wie user@local.nvtest.com.

image:ad2.png[ad_config]

Basis-DN: Dies ist ein Windows Active Director-Stammobjekt für die Benutzerauthentifizierung. Die Mindestanforderung für die Zugriffsberechtigung ist Lesen. Wie im obigen Beispiel gezeigt, ist das Objekt OU=IT,DC=local,DC=nvtest,DC=com nur für ein Benutzerkonto zulässig, das im Feld Benutzername definiert ist, um das Lesen zu ermöglichen.

image:ad3.png[ad_config]

Mit den oben genannten Einstellungen für den Benutzernamen und den Basis-DN ist {product-name} in der Lage, sich erfolgreich mit Windows Active Directory zu verbinden. Klicken Sie auf VERBINDUNG TESTEN, um sie zu überprüfen.

image:ad4.png[ad_config]

Name des Benutzers: Es ist erforderlich, für die Übereinstimmung NUR das Attribut sAMAccountName zu verwenden. Im folgenden Beispiel prüft {product-name}, ob sich der Benutzer ituser(CN=ituser,OU=IT,DC=local,DC=nvtest,DC=com) mit der Webkonsole {product-name} anmelden kann.

[NOTE]
====
{product-name} verwendet nicht die Werte der Attribute cn, displayName, dn, givenName, name oder userPrincipalName usw., um den Testbenutzer zu überprüfen.
====

image:ad5.png[ad_config]

Der letzte Teil ist die Rollenzuweisung für {product-name} für die Anmeldung an der Webkonsole.

image:ad6.png[ad_config]

Im obigen Beispiel muss die definierte Gruppe _d_s_itgroup in der Rolle {product-name} die Attribute member und sAMAccountType haben. Der Wert des Attributs sAMAccountType MUSS 268435456 lauten, d. h. die globale Sicherheitsgruppe, und der Login-Benutzername muss in den Mitgliederlisten enthalten sein.

image:ad7.png[ad_config]

Attribut des Gruppenmitglieds: Dies ist ein standardmäßiges Mitgliedsattribut für Windows Active Directory und wird, wie oben gezeigt, für die Rollenzuordnung verwendet.
Wenn alle oben genannten Anforderungen erfüllt sind, sollte sich der Windows Active Directory-Benutzer erfolgreich bei der Webkonsole {product-name} anmelden können.
