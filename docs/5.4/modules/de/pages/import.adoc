== Importieren einer Datei im CRD-Format über die Konsole oder API

{product-name} unterstützt das Importieren einer CRD-formatierten Datei von der Konsole aus. Dies ist jedoch nicht dasselbe wie die Anwendung in Kubernetes als benutzerdefinierte Ressourcendefinition (CRD).

Eine Datei im Format {product-name} CRD kann über die Konsole importiert werden, um die in der Datei angegebenen Sicherheitsrichtlinien (Regeln) festzulegen. Diese Regeln werden NICHT als "CRD"-spezifische Regeln importiert, sondern als reguläre "vom Benutzer erstellte" Regeln. Das bedeutet, dass diese Regeln wie andere Regeln über die Konsole oder die API geändert oder gelöscht werden können. Sie sind nicht als CRD-Vorschriften vor Änderungen geschützt.

Um von der Konsole aus zu importieren, gehen Sie zu Richtlinie -> Gruppen und wählen Sie Richtliniengruppe importieren.

image:4-3_Import_Policy.png[import]

[IMPORTANT]
====
Die importierten Regeln überschreiben alle bestehenden Regeln für die Gruppe.
====

Regeln, die mit den Kubernetes-CRD-Funktionen festgelegt werden, z. B. durch "kubectl apply my_crd.yaml", erstellen Regeln vom Typ CRD in {product-name}, die nicht über die Konsole oder API geändert werden können. Diese können nur geändert werden, indem die crd-Datei aktualisiert wird und die Änderung über Kubernetes angewendet wird.

Mögliche Anwendungsfälle für den Konsolenimport der Regeldatei sind:

* Erstmalige (einmalige) Konfiguration von Regeln für eine Gruppe oder Gruppen
* Migration von Regeln aus einer Umgebung in eine andere
* Erstellung von Regeln, deren Änderung über die Konsole oder die API erlaubt sein muss.
